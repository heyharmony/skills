# PRD Writer

> Turn customer conversations and product thinking into a stakeholder-ready Product Requirements Document — grounded in evidence, not assumptions.

## What It Does

Writes a complete PRD using a proven 8-section template: summary, contacts, background, objective with measurable key results, market segments, value proposition, solution spec, and release plan. When customer conversations live in Harmony, the PRD is built on real evidence — customer quotes, validated pain points, and competitive intelligence from actual calls — not guesswork.

## Who It's For

- **Product managers** writing specs for new features or products
- **Founders** formalizing product thinking before engineering kicks off
- **Engineering leads** who need clear requirements and success criteria
- **Product teams** restructuring messy requirements into a coherent document

## Use Cases

- Dozen customer calls done, need to formalize into a spec → PRD with customer evidence, measurable KRs, and phased release plan
- Feature idea approved by leadership, need to spec it out → Complete PRD with solution detail, assumptions flagged, and dependencies mapped
- Existing PRD is a mess and needs restructuring → Rewrite into 8-section template with gap analysis
- Linear/Jira tickets exist but no cohesive document → Synthesized PRD that connects tickets to customer problems and strategic goals

## Setup

Install and go. For the strongest PRDs, have customer conversations in Harmony — the skill pulls direct quotes, pain point evidence, and feature request language from your calls to ground every section in reality.

## What You Get

A structured PRD with: executive summary, stakeholder contacts, evidence-based background, SMART objectives with key results, validated market segments, value proposition with competitive positioning, detailed solution spec (features, UX flows, technical considerations, assumptions, risks), and a phased release plan with success criteria.
