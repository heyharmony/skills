# Apply Kaizen Method

> Transform frustration into incremental wins. Small, structured improvements compound into major gains.

## What It Does

Apply Kaizen Method runs a disciplined root-cause analysis on recurring problems—using the 5 Whys, Cause-and-Effect mapping, and impact scoring—then delivers a concrete 2-week improvement sprint plan. Each fix is small, testable, reversible, and tied to a success metric.

The skill pulls evidence from Harmony sources (Slack retros, Linear issue patterns, Jira trends, Notion docs) or from your description of a bottleneck. It surfaces the real system failure, not just the symptom, and proposes 3–5 incremental solutions scoped to ship this sprint.

## Who It's For

- **Operations/Engineering teams:** Fixing onboarding cycles, deployment friction, incident response delays.
- **Product/Design teams:** Reducing rework loops, accelerating design-to-dev handoff, clarifying requirements.
- **Sales/Support teams:** Speeding up ticket resolution, closing follow-up gaps, standardizing data entry.
- **Leadership/All-hands:** Lowering meeting load, clarifying decision paths, eliminating duplicate work.

Any team stuck in a repetitive pain loop that doesn't require external expertise or strategy changes.

## Use Cases

1. **"Why does customer onboarding take 3x longer than it should?"**
   → Kaizen reveals manual bottleneck; proposes self-serve template + automated reporting.

2. **"Our design reviews are always late. Why?"**
   → Uncovers missing Slack integration + unclear handoff timing; fixes both in 1 sprint.

3. **"Support queue is drowning. What's the real issue?"**
   → Data shows 60% of tickets are FAQ-able; proposes quick knowledge base + auto-response rules.

4. **"We keep having the same production incident."**
   → 5 Whys reveals monitoring gap; proposes alert config fix shipping this week.

## Connectors

- **Slack:** Surface team frustrations and recurring complaints from recent channels.
- **Linear:** Analyze issue history and cycle trends for pattern spotting.
- **Jira:** Pull sprint data, blocked tickets, and bug trends for process bottleneck signals.
- **Notion:** Reference existing process docs and past improvement initiatives.

## Setup

1. **Identify the pain point:** Use a Kaizen trigger phrase ("Why does X keep happening?" or "Continuous improvement around Y").
2. **Point to sources:** Share Slack channels, Linear cycles, or Notion docs with context.
3. **Define the metric:** What does success look like? (Faster, fewer tickets, higher adoption, lower cost?)
4. **Kick off the sprint:** Assign owners, run the 2-week plan, measure results weekly.

## What You Get

- **Problem statement** clearly defined with baseline metrics.
- **5 Whys analysis** drilling to root cause (not surface symptom).
- **Cause-and-Effect diagram** mapping all failure points.
- **Prioritized improvement list** (3–5 fixes ranked by impact).
- **2-week sprint plan** with owners, timelines, and success metrics.
- **Measurement framework** to prove the fixes worked.

**Turnaround:** 1 hour to analyze + deliver plan. 2 weeks to implement and measure.
