# Programmatic SEO

> Turn keyword patterns into hundreds of SEO-winning pages. One template. Infinite variations. Real organic traffic.

## What It Does

Programmatic SEO automates the creation of high-volume, templated pages that rank for long-tail keyword variations. Instead of manually writing 50 comparison pages or 200 integration guides, you design one template, source the data, and generate pages at scale.

This skill walks you through the entire process:
- **Identify** keyword patterns your competitors miss
- **Choose** a playbook (comparisons, integrations, locations, etc.)
- **Design** a template that adds unique value per page
- **Source** data from HubSpot, spreadsheets, and APIs
- **Build** internal linking architecture (hub-and-spoke)
- **Launch** with quality checks and indexation monitoring

The result: hundreds of pages that drive organic traffic, build site authority, and create natural internal linking. Fieldwork, a field service SaaS, launched 38 integration pages (targeting 9,200 monthly searches), indexed 90% within 2 weeks, and captured 8–12% CTR.

## Use Cases

1. **Comparison playbook**: Rank for "X vs Competitor A/B/C" searches. Fieldwork targets "Fieldwork vs Jira", "Fieldwork vs ServiceMax", "Fieldwork vs Vantage"—40 comparisons from one template.

2. **Integration playbook**: Answer "How do I integrate X with Y?" Zapier, Make, Stripe, and Square all have 200+ integration pages. Your SaaS can too.

3. **Location playbook**: "Service available in [city]" or "[industry] solutions in [region]". HubSpot's industry guides target 100+ industries; same model works for locations.

4. **Glossary playbook**: "What is [term]?" Define niche concepts in your industry. Monetizes question-based searches and builds topical authority.

5. **Directory/partnership playbook**: "Top [category] tools", "[Industry] alternatives". Aggregate lists with unique analysis (your use cases, integrations, pricing).

6. **Persona playbook**: "[Role] guide to [topic]". Engineer's guide to API monitoring, marketer's guide to CDP evaluation—angle each page to one persona.

## Setup

1. **Audit your data**: Pull your top 100 keywords from Google Search Console. Identify 3–5 playbook opportunities (comparisons, integrations, locations).

2. **Validate demand**: Use keyword research tool to confirm patterns have 50+ monthly searches each. Cross-check with Slack (team asks these Qs?) and HubSpot (customers mention these?).

3. **Choose one playbook**: Start with the highest-volume, lowest-competition pattern. Prove the model before expanding to 3–4 playbooks.

4. **Design your template**: Map unique value per page. Example: each comparison page must include feature matrix, ROI analysis, migration path, customer use case, pricing (not just name swaps).

5. **Source data**: Combine proprietary (HubSpot customer data, Slack signals) with public (competitor sites, G2 reviews). Structure in a spreadsheet.

6. **Set up tracking**: Create a Notion page inventory and Linear ticket. Track indexation in Google Search Console.
