# Site Architecture Planner

> **Build websites users can navigate intuitively. Map your page hierarchy, plan your navigation, and link your content strategically—before you build.**

## What It Does

This skill turns chaos into clarity. You describe your site's purpose and content; the skill walks you through designing a logical page hierarchy, naming URLs consistently, building smart navigation systems, and creating an internal linking strategy that serves both users and search engines.

Whether you're launching a new site, restructuring an existing one, or untangling a sprawling content library, this skill provides a repeatable framework for site architecture that scales.

## Who It's For

- **Marketing Directors** planning website redesigns or launches
- **Content Strategists** organizing large content libraries (docs, blog, knowledge base)
- **Product Managers** designing information architecture for SaaS platforms
- **Growth Teams** improving navigation to reduce bounce rate and boost conversion
- **Support Leaders** fixing confusing site structures that drive ticket volume
- **Startup Founders** building their first website without architectural mistakes

## Use Cases

1. **Website Redesign** — You're redesigning your site and need a clear blueprint before dev starts
2. **Content Migration** — Moving to a new platform or consolidating duplicate pages; you want a better structure
3. **Scaling Content** — Your blog, docs, or product features have grown too large; you need a system
4. **Navigation Audit** — Users can't find things; analytics show high bounce rates on key pages
5. **SEO Improvement** — Internal linking and page depth are hurting rankings; you need a strategic overhaul
6. **Conversion Optimization** — Your funnel is leaky; you're restructuring pages to guide users toward key actions

- **HubSpot** — Pull page performance data and visitor flow patterns
- **Notion** — Maintain a living site architecture document for your team
- **Linear / Jira** — Create dev tickets for navigation updates and site restructuring work

## Setup

1. Gather your site's business goals, key metrics, and target user personas
2. List your current pages or plan new pages you'll create
3. Run the skill to map your hierarchy, design navigation, and plan internal linking
4. Use the output sitemap and URL structure to align your team and brief developers
5. Implement the architecture; update docs and dev tickets in your project tracker

## What You Get

- **Page Hierarchy Diagram** — ASCII tree or Mermaid visual showing levels, nesting, and relationships
- **URL Mapping Table** — Every page's URL, level, and priority in a clear reference table
- **Navigation Specification** — Header, footer, sidebar, and breadcrumb patterns (exact structure)
- **Visual Sitemap** — Mermaid diagram showing page connections and internal linking paths
- **Internal Linking Strategy** — Hub-and-spoke recommendations, contextual link placement, SEO priorities
- **Migration Checklist** — If restructuring, a plan for redirects, archived pages, and rollout timeline
