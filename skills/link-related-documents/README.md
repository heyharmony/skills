# Link Related Documents

> Map hidden connections between documents, meetings, and threads to build a navigable knowledge network and surface gaps.

## What It Does

Transforms scattered docs and conversations into a connected knowledge graph. The skill:

- **Extracts entities** — People, projects, decisions, dates, and topics from across your org
- **Finds connections** — Direct references, shared entities, temporal relationships, and causal chains
- **Scores relationships** — Weights connections by strength (1–5 scale) based on evidence
- **Identifies gaps** — Orphaned docs, missing documentation, undocumented decisions
- **Maps the network** — Creates visual diagrams and searchable catalogs showing how everything relates
- **Recommends actions** — Suggests which docs to link, create, archive, or update

Works from Notion, Google Drive, Slack, and Confluence. Outputs a complete knowledge network report.

## Who It's For

- **Compliance & Legal Teams** — Trace decision lineage, find all related docs for audits
- **Knowledge Managers** — Build org wikis and knowledge bases, prevent duplicate work
- **New Team Members** — Understand how decisions, projects, and docs relate during onboarding
- **Project & Program Managers** — See how past initiatives connect to new planning
- **Operations & Founder** — Track what decisions were made, who made them, and where the rationale is documented

## Use Cases

- **Post-project review:** Connect all docs from a major initiative into one navigable map
- **Onboarding prep:** Build knowledge graph so new hires see context and decision history
- **Audit trail:** Find all docs related to a decision, regulation, or compliance event
- **Team transition:** When someone leaves, map their projects and decisions to preserve knowledge
- **Duplicate prevention:** Discover whether a project/doc already exists elsewhere
- **Strategic planning:** See how current work connects to past initiatives and strategy

## Connectors

| Connector | Used For |
|-----------|----------|
| **Notion** | Pull pages, databases, wikis, and decision records |
| **Google Drive** | Surface related docs, sheets, presentations, and shared folders |
| **Slack** | Find threads that reference or discuss document topics |
| **Confluence** | Pull wiki pages and knowledge base articles for cross-linking |

## Setup

1. **Define scope:** Topic (compliance initiative), time period (Q1 2026), person (author), or project
2. **Gather documents:** Pull from Notion, Google Drive, Slack, or Confluence
3. **Choose output format:** Visual map (Miro/Lucidchart), Notion database, or written report
4. **Identify stakeholders:** Who needs to review the knowledge map? (3–5 people ideal)

## What You Get

- **Knowledge Network Report** (Notion page or Google Drive doc) with:
  - Visual network diagram (nodes + edges showing connections)
  - Document catalog (searchable table of all docs with metadata)
  - Connection index (all links scored by type and strength)
  - Gap analysis (orphaned docs, missing documentation, risks)
  - Prioritized recommendations (what to link, create, or archive)

- **Optional:** Interactive dashboard (Notion with filters), embedded Lucidchart diagram, timeline visualization
