# Onboarding-CRO

**Optimize post-signup user activation flows to reduce time-to-value and increase activation rates.**

## What This Skill Does

This skill analyzes your conversation history—customer calls, support tickets, sales discussions, team chat—to identify why users sign up but don't activate. It uncovers the real "aha moment" users need to experience, maps where they get stuck, and recommends specific onboarding improvements backed by actual user feedback.

Activation is not guaranteed after signup. This skill helps you build a flow that guides users from signup → first meaningful action → activation faster.

## When to Use This

- Your activation rate is plateauing and you need to understand the real friction points
- You're rebuilding onboarding and want to prioritize changes that matter most
- Support and sales calls reveal the same user struggles—and you want to fix it at the source
- You're testing different onboarding patterns (guided setup, product-first, value-first)
- You have stalled users and need a re-engagement strategy
- You want to measure onboarding success with clarity on what to optimize next

## What You Get

A structured analysis with:
- Clear diagnosis of current activation bottlenecks (grounded in conversation insights)
- The specific moment users need to experience to feel confident (the aha moment)
- Actionable recommendations ranked by impact and effort
- Success metrics so you know if your changes worked
- A rollout roadmap with timeline and measurement approach

## How It Works

1. Connect your conversation sources (customer calls, support tickets, Slack, sales notes)
2. Run the skill to analyze what users actually need during onboarding
3. Receive recommendations: quick wins, medium-term improvements, and long-term changes
4. Use the roadmap to prioritize and measure impact

## Connected Data Sources

- **HubSpot**: Activation metrics, user lifecycle data, cohort analysis
- **Slack**: Team discussions about user activation, customer feedback, support challenges
- **Notion**: Onboarding documentation, product specs, past experiments
- **Linear**: Dev tickets for onboarding feature requests and improvements
- **Intercom**: Onboarding messages, user engagement data, re-engagement campaigns

## Typical Output

A concise recommendations document (2-3 pages) with current state, identified aha moment, gap analysis, phased improvements, and success metrics—ready to share with product, design, and dev teams.
