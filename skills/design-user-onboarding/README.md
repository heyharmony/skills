# Design User Onboarding

> The best onboarding doesn't feel like onboarding—it feels like using the product.

## What It Does

First impressions determine retention. This skill maps the journey from signup to first value (time-to-value), identifies where users get stuck, and designs flows tailored to different user personas. It prioritizes reducing friction ruthlessly—every step costs users—while celebrating small wins along the way. The output is a clear onboarding funnel with drop-off analysis, persona-specific flows, and a roadmap to get more users to their "aha moment" faster.

## Who It's For

- **Product Managers**: Need to understand why users drop off during setup and how to improve week-1 activation. Pain point: 60% of signups never create their first artifact or reach core value.

- **Designers/UX Leads**: Designing first-time user experience. Pain point: Balancing simplicity with showing enough features to convey capability.

- **Growth Leaders**: Improving activation rate and reducing churn at the top of the funnel. Pain point: Acquisition is cheap but conversion to active users is low.

- **Founders**: Proving early product-market fit. Pain point: High signup count but low activation; unclear if it's product problem or onboarding problem.

- **Operations/Support Teams**: Seeing repetitive setup questions. Pain point: Same questions repeat because onboarding is unclear or missing critical guidance.

- **Analytics Teams**: Measuring onboarding effectiveness. Pain point: No clear metrics for success; conflating completion rate with actual value delivery.

## Use Cases

- **Your activation rate is below 40%** → Map your onboarding funnel to find where users drop off; redesign that step

- **Different user types have very different needs** → Design 2-3 persona-specific flows that guide each type to their unique "why you need this" moment

- **Your onboarding is too complex** → Audit every step; remove information that isn't needed for week-1 value; reveal complexity progressively

- **You're considering feature tours or tooltips** → First verify if the issue is actual UX complexity or just information overload; fix UX before adding guides

- **Signup is strong but activation is weak** → Diagnose which moment in onboarding doesn't deliver value; redesign that moment to show core benefit earlier

- **You're redesigning onboarding** → Test new flow against current flow; measure which persona benefits most from which variant

## Connectors

Works standalone, but richer with:

- **Google Analytics**: Measure onboarding funnel drop-off at each step; identify which step loses most users

- **Session Recordings (Clarity/Hotjar)**: Watch how users actually navigate onboarding; identify unexpected friction points visually

- **Slack**: Monitor user feedback and support tickets about confusion during setup or early feature use

- **Intercom**: Target onboarding messages in-app; trigger tooltips and guides at moments when users need help without over-explaining

## Setup

No setup required. Have your product, your recent signup data, and ideally some notes on why users have stopped or struggled. If you have analytics (Google Analytics, Amplitude, etc.), you'll measure funnel drop-off precisely. If not, this skill will help you design the measurement framework to build going forward.

## What You Get

- Baseline activation analysis: current week-1 activation rate, time-to-value, and comparison to industry benchmarks

- User personas: 2-4 distinct personas with their first-use goals and assumptions

- Current onboarding funnel: every step mapped with completion rate and key friction points

- Dead zone identification: the exact moments where users drop off most, with root causes from user research

- Persona-specific flow designs: 2-3 tailored flows that address different user types without overwhelming all of them

- Empty state coaching examples: how to use blank pages as teaching moments rather than confusing dead ends

- Progressive disclosure strategy: what information to show when, to balance simplicity with capability

- Celebration moments: micro-wins and progress markers that build momentum through onboarding

- 30-day optimization roadmap: prioritized experiments to fix dead zones and increase activation rate

## Key Insight

The best products make onboarding invisible. Users don't think they're "onboarding"—they're just using the product and discovering why it's valuable. Every additional step you add loses users. Every moment of confusion is an exit point. Your goal is to get users to their first win (atomic action) in under 5 minutes, so they understand the core benefit and want to come back. Personas need different paths. Celebrate progress to build momentum. And measure activation not by "completed onboarding" but by "reached core behavior that predicts retention."
