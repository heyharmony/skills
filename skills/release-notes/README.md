# Release Notes Writer

> Turn a pile of tickets into release notes your customers actually want to read — benefits first, jargon never.

## What It Does

Transforms Jira/Linear tickets, PRDs, Git logs, or sprint conversations into polished, user-facing release notes. Translates technical changes into human benefits, categorizes by impact (new features, improvements, fixes, breaking changes, deprecations), and matches your product's tone — whether that's developer-precise, B2B-professional, or consumer-friendly. When team conversations exist in Harmony, it pulls demo language and customer context to write entries that connect features to real user problems.

## Who It's For

- **Product managers** writing customer-facing changelogs after each release
- **DevRel / Developer advocates** writing technical release notes for API products
- **Marketing teams** turning shipped features into announcement content
- **CS and sales teams** who need internal release notes to stay ahead of customer questions

## Use Cases

- Sprint just finished, 15 Linear tickets shipped → Categorized release notes with hero feature, improvements, and bug fixes
- Major version release with breaking changes → Release notes with prominent migration callout, timeline, and step-by-step guide
- Nothing exciting shipped, mostly tech debt and fixes → "Stability and performance" release notes that frame reliability as a feature
- Need both customer-facing and internal versions → External release notes plus internal sales/CS guide with talking points per feature

## Setup

Install and go. Provide tickets, PRDs, Git logs, or just describe what shipped. For richer context, have sprint review or demo meetings in Harmony — the skill uses team discussions about features to write better benefit-focused descriptions.

## What You Get

Structured release notes with: one-line release summary, categorized entries (new features, improvements, bug fixes, breaking changes, deprecations), benefit-first writing for every entry, audience-appropriate tone, and optional internal version with sales/CS talking points.
